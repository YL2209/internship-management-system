# 实习过程管理系统 — Docker 部署版

> 一款轻量级的实习过程数据管理平台，帮助实习生集中记录、查看和分析个人实习经历，支持 Docker 一键部署与 Web 可视化操作。
> 本项目基于 ckkkf/sign-sign-in (https://github.com/ckkkf/sign-sign-in) 改造，提供浏览器端配置管理与签到控制界面。
> 本项目代码全部由AI进行编写

---

## 项目简介

本系统专为需要管理实习期间各类数据的学生设计，提供**实习日志（周记/月记）撰写与提交**、**考勤记录查询**、**补签申请**、**数据统计概览**等核心功能。无需手动编辑配置文件，所有操作均在浏览器中完成，适合个人或小团队快速搭建私有实习数据仓库。

系统本身不采集或自动生成任何数据，所有数据均需由用户通过官方实习平台获取后录入，或由系统提供辅助提交接口（需用户主动授权）。其核心价值在于将分散的实习过程信息统一归档，方便回顾与导出。

---

## 功能特性

- ✅ **实习日志管理**：在线撰写、修改、提交周记与月记，支持字数统计与素材库保存。
- ✅ **考勤记录查看**：展示历史考勤打卡记录（数据来源为用户所绑定的实习平台），支持按时间、状态筛选，并提供签到详情、排行榜与补签操作。
- ✅ **签到管理（新版）**：集成普通签到/签退、拍照签到、补签申请、打卡详情查询及月度排行榜。
- ✅ **数据概览**：自动统计日志数量、考勤成功率、近期趋势，并以图表形式呈现。
- ✅ **Web 管理后台**：直观的图形界面，无需命令行操作，所有功能均在浏览器中完成。
- ✅ **用户认证**：内置管理员账号，支持密码修改与登录历史审计，保障数据隐私。
- ✅ **Docker 一键部署**：镜像即开即用，数据持久化存储，支持配置热加载。
- ✅ **智能位置模拟**：支持动态坐标抖动（每日漂移 + 高斯噪声），增强签到行为自然性，降低风控风险。
- ✅ **缓存管理**：内置缓存系统（排行榜、签到详情、周记列表），可独立设置过期时间，提升响应速度。

## 快速开始

### 1. 获取项目

```bash
git clone https://github.com/YL2209/internship-management-system.git
cd internship-management-system
```

### 2. 准备配置文件

复制示例配置文件并根据实际情况修改：

```bash
cp config.default.json config.json
```

> 您需要填写实习平台的认证信息（如 `openId`、`unionId`、`encryptValue`、`sessionId` 等）。这些参数可从实习平台的官方小程序或 Web 端通过合法方式获取（例如登录后的接口响应）。具体获取方法请参考实习平台的技术文档或联系平台管理员。

### 3. 启动服务

使用 Docker Compose 一键启动所有服务：

```bash
docker compose up -d
```

访问 `http://localhost:5000` 即可进入 Web 管理后台。

### 4. 登录后台

- 默认账号：`admin`
- 默认密码：`admin123`

**首次登录后请立即修改密码**（通过 Web 界面右上角的“账户安全”页面）。

## Web 管理后台使用指南

### 仪表盘
- 显示实习日志总数、考勤记录总数、成功率、定时任务状态等概览信息。
- 展示近 30 天考勤趋势、热力日历、时段分布、周记/月记统计等图表。

### 实习日志管理
- **素材库**：存储已撰写或待提交的周记/月记草稿，支持增删改查，方便复用。
- **提交记录**：查看通过本系统提交至实习平台的日志历史（成功/失败状态）。
- **撰写与提交**：选择周记或月记类型，填写标题、正文、起止日期，提交至实习平台（需配置有效认证信息）。

### 签到管理（新版集成）
- **签到详情**：查看当天打卡状态（签到/签退时间、地址、坐标），并可直接执行普通签到或拍照签到。
- **排行榜**：查看指定月份的实习工时排行榜，支持缓存加速。
- **补签操作**：自动拉取可补签日期，选择日期后自动填充补签类型、地址、随机补签理由，一键提交。

### 考勤记录
- 查看历史考勤打卡记录，支持按日期、类型（签到/签退）、成功/失败筛选。
- 记录包含时间、地址、GPS 坐标、设备信息等详细信息。

### 系统配置
- **认证方式**：支持“自动模式”（使用授权码换取会话）与“手动模式”（直接填写固定参数）。
- **位置与设备**：设置经纬度、地图服务商、设备型号、User‑Agent 等环境参数。
- **位置漂移设置**：可配置每日漂移最小/最大距离（米），以及单次抖动半径，实现动态坐标变化。
- **定时任务**（可选）：可配置签到/签退的自动执行计划，但默认关闭，建议手动操作。
- **缓存控制**：可独立设置排行榜、签到详情、周记列表的缓存过期时间。

### 账户安全
- 修改管理员密码。
- 查看登录历史（含 IP 与地理位置）。

### 备份管理
- 手动创建配置备份，支持备注、查看、对比、恢复与删除。
- 支持自动备份（可配置间隔与保留数量）。

### 缓存管理
- 查看各缓存（排行榜、签到详情、周记列表）的统计信息、条目详情及实际数据。

## 配置说明

系统通过 `config.json` 进行核心配置，部分敏感参数可通过 `.env` 环境变量覆盖。以下为常用字段说明：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `auth_mode` | 认证模式：`"auto"`（自动登录，需提供 `code`）或 `"manual"`（手动模式，需填写固定参数） | `"manual"` |
| `code` | 授权码（自动模式使用） | `""` |
| `openId` / `unionId` | 用户标识（手动模式必填） | `""` |
| `encryptValue` / `sessionId` | 会话凭证（手动模式必填，通常有效期为 24 小时） | `""` |
| `location` | 经纬度（用于考勤打卡定位） | `{"latitude":"39.916527","longitude":"116.397128"}` |
| `locationJitterMeters` | 单次高斯抖动的最大半径（米） | `100` |
| `dailyDriftMinMeters` | 每日基准点漂移的最小距离（米） | `5` |
| `dailyDriftMaxMeters` | 每日基准点漂移的最大距离（米） | `10` |
| `device` | 设备信息（品牌、型号、系统、平台） | 示例值 |
| `userAgent` | 模拟的 User‑Agent 头 | 微信小程序 UA |
| `schedule` | 定时任务配置（默认关闭） | `{"enabled": false, ...}` |
| `auto_backup` | 自动备份策略 | `{"enabled": false, ...}` |
| `cache_ttl` | 通用缓存过期时间（秒） | `86400` |
| `cache_ttl_ranking` | 排行榜缓存过期时间（秒） | `3600` |
| `cache_ttl_clock_detail` | 签到详情缓存过期时间（秒） | `7200` |
| `cache_ttl_journal_list` | 周记列表缓存过期时间（秒） | `43200` |

> **注意**：`sessionId` 等凭证具有时效性，过期后需重新获取并更新配置。获取方式请参考所使用实习平台的官方开发者文档。

## Docker 部署详解

### 环境变量（`.env`）

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `WEB_PORT` | Web 服务宿主机映射端口 | `5000` |
| `ADMIN_PASSWORD` | 管理员明文密码（首次启动自动哈希） | 必填 |
| `JWT_SECRET` | JWT 签名密钥（生产环境应设置） | 随机生成 |
| `LOG_LEVEL` | 日志级别（DEBUG/INFO/WARNING/ERROR） | `INFO` |
| `TZ` | 时区 | `Asia/Shanghai` |

### 常用命令

```bash
# 启动所有服务（守护进程 + Web）
docker compose up -d

# 查看日志
docker compose logs -f

# 停止服务
docker compose down

# 修改配置后重启
docker compose restart
```

### 数据持久化

以下目录通过 Docker Volume 挂载，容器重启或升级不会丢失数据：
- `./config.json` — 配置文件（读写）
- `./cache` — 会话缓存
- `./logs` — 日志文件
- `./backups` — 配置备份
- `./journals` — 周记/月记素材库

## 风控规避说明

部分实习平台会通过规则引擎检测自动化行为（如异常 IP、规律性坐标、重复设备指纹等），以下为本系统内置的规避措施及用户应注意事项：

### ✅ 已内置的防护措施

| 风控规则 | 防护方式 | 效果 |
|----------|----------|------|
| **签到地址异常规律** | 每日基准点漂移 + 高斯抖动，每次签到坐标随机偏移（可配置半径） | 避免坐标完全重复，模拟真实 GPS 误差 |
| **签到设备异常** | 设备信息、User‑Agent 固定填写（与抓包一致） | 避免频繁切换设备导致风控 |
| **时间规律性** | 定时任务支持 `random_window` 随机偏移（如 ±10 分钟） | 避免每天同一秒打卡 |
| **会话凭证复用** | 支持缓存会话凭证（24 小时有效），减少频繁登录 | 降低因登录触发的异常检测 |
| **拍照图片防护** | 上传图片自动清除 EXIF 信息（GPS、拍摄时间等），防止隐私泄露 | 防止图片携带真实位置信息 |

### ⚠️ 用户需自行注意的高风险项

1. **IP 身份异常**  
   若将本系统部署在云服务器上，IP 会被识别为机房/IDC IP，**极易触发风控**。  
   **建议**：  
   - 使用家庭宽带、4G/5G 手机热点等**住宅 IP** 出口  
   - 或通过合规的住宅代理服务转发签到请求  

2. **外勤地址超范围 / 连续外勤**  
   签到坐标不应长期大幅偏离实习单位设定的打卡范围（通常为地理围栏）。  
   **建议**：  
   - 将配置中的 `location` 经纬度填写在单位围栏内  
   - `locationJitterMeters` 抖动半径保持合理值（建议 ≤ 200 米）  

3. **签到图片重复使用**（适用于拍照签到）  
   如果启用拍照签到功能，**切勿重复使用同一张照片**，否则会被识别为“签到图片重复使用”。  
   **建议**：  
   - 每次使用不同的真实照片  
   - 或对照片进行轻微调整（裁剪、亮度微调等）  

4. **凭证时效管理**  
   `sessionId` 等凭证存在有效期（通常 24 小时），过期后系统会自动清除缓存并提示重新获取。请及时更新配置，避免因凭证失效导致操作失败。

### 📋 风控规则对照表

| 规则 | 状态 | 说明 |
|------|------|------|
| IP身份/用途异常 | 🔴 **高风险** | 必须使用住宅 IP，避免云服务器直连 |
| 签到地址异常规律 | 🟢 **已规避** | 每日漂移 + 高斯抖动 |
| 连续3天外勤签到 | 🟢 **已规避** | 锚点需在围栏内 |
| 外勤地址超正常范围 | 🟢 **已规避** | 不频繁大幅修改锚点 |
| 签到图片重复使用 | 🟡 **可配置** | 若启用拍照签到，需自行保证图片不重复 |
| 签到设备异常 | 🟢 **已规避** | 设备指纹固定 |
| IP聚合签到异常 | 🔴 **高风险** | 必须使用独立 IP，避免共享 |

## 注意事项

- **认证凭证**：`sessionId` 等参数具有时效性（通常 24 小时），过期后需更新配置。请妥善保管，避免泄露。
- **位置数据**：GPS 坐标用于考勤打卡，可启用随机抖动（`locationJitterMeters`、`dailyDriftMin/Max`）以模拟真实场景，但应控制在合理范围内（默认最大 500 米）。
- **设备信息**：`device` 和 `userAgent` 需与实习平台环境匹配，否则可能导致请求被拒。
- **定时任务**：该功能默认关闭，启用后需确保认证凭证长期有效，否则任务将失败。建议仅在有持续有效会话的情况下使用。
- **安全建议**：生产环境务必修改默认管理员密码，并设置 `JWT_SECRET` 为强随机字符串。
- **图片隐私**：拍照签到上传的图片会经过去除 EXIF 处理，不会保留拍摄位置等隐私信息。
